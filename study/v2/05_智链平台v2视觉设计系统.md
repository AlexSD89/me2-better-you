  secondary-500:
    - 次要按钮和操作
    - 高亮信息和标签
    - 数据可视化

    - 特殊状态和提醒
    - 创新功能突出显示
    - 引导性元素

  主文本: 对比度 ≥ 7:1 (AAA级)
  次要文本: 对比度 ≥ 4.5:1 (AA级)

  装饰元素: 对比度 ≥ 2:1

  lg: 24px (标准功能图标)
  xl: 32px (大型功能图标)
  2xl: 48px (特殊场景图标)

图标颜色规范:
  默认状态: var(--cloudsway-text-muted)

## 📱 响应式设计原则

### 移动优先设计
```yaml
设计原则:
  核心理念: Mobile First - 从最小屏幕开始设计
  内容优先: 确保核心功能在所有设备上都可用
  性能优化: 移动端加载速度优先考虑
  触摸友好: 足够大的触摸目标 (最小44px)

断点策略:
  320px-639px (Mobile):
    - 单列布局
    - 堆叠导航
    - 简化操作流程
    - 重点突出核心功能
    
  640px-767px (Large Mobile):
    - 开始引入侧边栏
    - 两列网格布局
    - 更丰富的交互元素
    
  768px-1023px (Tablet):
    - 经典的1:3:6布局开始出现
    - 侧边栏导航
    - 卡片网格优化
    
  1024px+ (Desktop):
    - 完整的桌面体验
    - 复杂的多列布局
    - 高级交互功能
```

## 🎯 品牌应用规范

### Logo使用规范
```yaml
Logo设计元素:
  主Logo: 包含图标和文字的完整标识
  图标Logo: 仅图标部分，用于小尺寸场景
  文字Logo: 仅文字部分，用于特殊场景

使用规范:
  最小尺寸: 24px (图标) / 120px (完整Logo)
  安全区域: Logo周围至少保留Logo高度的1/2空间
  颜色使用: 
    - 深色背景: 白色或主色版本
    - 浅色背景: 深色或主色版本
    - 彩色背景: 白色版本

禁用场景:
  - 不得拉伸变形
  - 不得更改颜色比例
  - 不得添加阴影或特效
  - 不得在低对比度背景使用
```
